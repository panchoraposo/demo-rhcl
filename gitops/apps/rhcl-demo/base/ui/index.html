Role separation — infra owner vs platform vs developer
One click prints checks that map outcomes to responsibilities. Great to explain “who owns what” in policy lifecycle.

Zero-trust by default

The Gateway is deny-all by default. Each role applies policies at a different layer to progressively allow only what is needed.

Infrastructure owner

Owns the entrypoint and transport security.
Gateway + listener TLSPolicy + cert-manager OpenShift Route passthrough

Platform engineer

Owns global guardrails and safe defaults.
Gateway AuthPolicy (deny-all) UI allow override

Developer

Owns route-level protection for the app and APIs.
API key AuthPolicy RateLimitPolicy OIDCPolicy + JWT AuthPolicy
